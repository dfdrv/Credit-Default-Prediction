## Задача

Бинарная классификация кредитного дефолта:

| Значение | Класс |
|---|---|
| `0` | Займ выплачен |
| `1` | Займ не выплачен (дефолт) |

Целевая метрика — **ROC-AUC**, выбор моделей и стратегии валидации ориентирован именно на неё (обучение на `predict_proba`, а не на жёсткую классификацию).

## Данные

| | Строк | Признаков |
|---|---|---|
| `shift_ml_2026_train.csv` | 1 210 779 | 109 (включая таргет) |
| `shift_ml_2026_test.csv` | 134 531 | 108 |

- Доля положительного класса (дефолт) — **≈20%**, умеренный дисбаланс.
- 88 числовых признаков, 19 категориальных.
- Категориальные признаки сильно различаются по кардинальности: от бинарных (`срок_займа`, `рейтинг`) до сверхвысококардинальных (`профессия_заемщика`, >270 тыс. уникальных значений).
- Пропуски присутствуют в **69 из 109** признаков, местами доля пропусков превышает 90%. Пропуски трактуются как потенциально информативный сигнал (отсутствие продукта/события у клиента), а не как дефект данных, и намеренно не импутируются — LightGBM и CatBoost обрабатывают `NaN` нативно.

## Пайплайн решения

```
train/test CSV
     │
     ▼
EDA (типы, пропуски, распределения, корреляции с таргетом)
     │
     ▼
Препроцессинг: конкатенация train+test → чистка признаков →
   date-фичи → LabelEncoding → разбиение обратно
     │
     ▼
Level-1 модели (5-fold Stratified CV, OOF-предсказания):
   LightGBM · CatBoost · LogisticRegression(L2) · ExtraTrees
     │
     ▼
Simple weighted ensemble (LightGBM + CatBoost)
     │
     ▼
Level-2 stacking: мета-модель LightGBM(depth≤3) на OOF + mean/std/min/max
     │
     ▼
Pseudo-labeling (осторожный, только при подтверждённом приросте OOF AUC)
     │
     ▼
Выбор лучшего варианта по OOF ROC-AUC → submission.csv / submission.zip
```

## EDA — ключевые находки

- **Дисбаланс классов** (~1:4) → стратифицированный K-Fold, метрика вероятностная, oversampling/undersampling не применялся — бустинги устойчивы к такому уровню дисбаланса.
- **Пропуски** сконцентрированы в признаках кредитной истории, агрегатах по счетам/займам и временных признаках; часто отражают отсутствие соответствующего продукта у клиента, а не ошибку сбора данных.
- **Числовые признаки** (сумма займа, лимиты, обороты) имеют выраженную правую асимметрию и выбросы → жёсткая нормализация/логарифмирование сознательно не применялись, поскольку основные модели — деревья, нечувствительные к монотонным преобразованиям и масштабу.
- **Корреляция с таргетом** у отдельных числовых признаков умеренная (топ значения в диапазоне 0.25–0.07) — сильной линейной связи ни у одного признака нет, что подтверждает целесообразность нелинейных моделей (бустинг) вместо линейных.

Эти наблюдения напрямую определили архитектурные решения ниже.

## Feature Engineering и препроцессинг

1. **Объединение train/test** во временный общий датасет (флаг `_is_train`) для идентичной обработки признаков.
2. **Удаление неинформативных признаков**:
   - константные (≤1 уникальное значение);
   - полностью пустые (100% NaN);
   - категориальные с ультравысокой кардинальностью (>10 000 уникальных значений, например `профессия_заемщика`) — исключены как источник шума/переобучения при отсутствии target-агрегации.
3. **Дата первого займа** (`дата_первого_займа`) разложена на три числовых признака: месяц, год и количество месяцев с момента первого займа до текущей даты.
4. **Кодирование категориальных признаков** — `LabelEncoder` (без OHE, ввиду высокой кардинальности части признаков и использования деревьев, для которых порядковое кодирование не создаёт ложной линейной семантики).
5. Пропуски не импутируются для бустингов; для LogReg/ExtraTrees заполняются нулями (`np.nan_to_num`), для LogReg дополнительно применяется `StandardScaler`.

## Моделирование

Валидация — **Stratified 5-Fold CV** (`SEED=42`) для всех моделей, с сохранением OOF-предсказаний для последующего стекинга.

### Level-1 модели

| Модель | Роль | Ключевые гиперпараметры |
|---|---|---|
| **LightGBM** | Основная модель | `learning_rate=0.05`, `num_leaves=63`, `n_estimators=3000` + early stopping (150), `subsample=colsample=0.8`, `reg_alpha=0.1`, `reg_lambda=1.0` |
| **CatBoost** | Альтернативная/комплементарная модель | `iterations=3000`, `learning_rate=0.05`, `depth=6`, `l2_leaf_reg=3`, `bootstrap_type=Bernoulli`, `subsample=0.8`, early stopping (150) |
| **LogisticRegression (L2)** | Линейный baseline / диверсификация ансамбля | `C=1.0`, `solver=lbfgs`, признаки предварительно масштабированы |
| **ExtraTreesClassifier** | Диверсификация ансамбля | `n_estimators=500`, `max_depth=12`, `min_samples_leaf=50`, `max_features='sqrt'` |

Бустинги выбраны как основа решения из-за эффективности на больших табличных данных и нативной поддержки пропусков/категорий высокой кардинальности. LogReg и ExtraTrees добавлены не ради собственного качества, а чтобы снизить корреляцию ошибок Level-1 моделей перед стекингом.

## Ансамблирование

Два подхода, оба оцениваются по OOF ROC-AUC:

1. **Simple weighted ensemble** — взвешенное среднее LightGBM и CatBoost, вес подбирается перебором по сетке `w ∈ [0, 1]` с шагом 0.05 на OOF.
2. **Stacking (Level-2)** — мета-модель LightGBM с жёстко ограниченной сложностью (`max_depth=3`, `num_leaves=7`, усиленная L1/L2-регуляризация), обучаемая на 8 мета-признаках:
   - 4 OOF-предсказания Level-1 моделей;
   - mean / std / min / max по этим предсказаниям.

   Малая глубина и малое число признаков — осознанное решение против переобучения мета-модели на OOF-сигналах.

## Pseudo-labeling

Осторожный эксперимент поверх стекинга:

- объекты теста с `proba ≥ 0.98` или `proba ≤ 0.02` считаются «уверенными» и добавляются в обучение с пониженным весом `0.3`;
- мета-модель переобучается на расширенной выборке (реальные данные + pseudo-labels) в том же 5-Fold CV;
- решение о включении в финальный submission принимается **только если pseudo-labeling дал прирост OOF ROC-AUC** относительно стекинга без него — иначе эксперимент помечается как неудачный и отбрасывается.

## Итоговые результаты

Финальный вариант выбирается автоматически — как модель/ансамбль с максимальным OOF ROC-AUC среди:

`LightGBM` · `CatBoost` · `LogReg (L2)` · `ExtraTrees` · `Simple Ensemble` · `Stacking` · `Stacking + Pseudo-Labeling` (при подтверждённом улучшении)

Ожидаемая иерархия качества: базовые бустинги дают сильный baseline → простой ансамбль улучшает результат за счёт комплементарности LightGBM/CatBoost → стекинг даёт дополнительный прирост за счёт адаптивного комбинирования сигналов → pseudo-labeling используется только при подтверждённом улучшении.

По итогам выбранная модель формирует:

- `submission.csv` — колонки `ID`, `Proba`;
- `submission.zip` — архив с `submission.csv` для сдачи в систему соревнования.

В ноутбуке также логируются контрольные статистики для воспроизводимости (seed, число фолдов, размерности выборок, состав ансамбля, итоговый OOF AUC, среднее/std предсказанных вероятностей).


- Blending с весами, оптимизированными не только для пары LightGBM/CatBoost, но и по всем Level-1 моделям.
